OCI AI Agent Prototype

A prototype that scans historical Jira tickets to suggest workarounds or route to SMEs, aimed at faster issue resolution. Below is a lightweight architecture and a few notes on evaluation.

High-level Architecture

Tickets are indexed into a vector store with minimal PII. A retrieval component selects the top-k similar tickets and summarizes prior fixes. The agent then proposes: (a) likely workaround, (b) candidate service owners/SMEs, and (c) confidence signals such as frequency-of-success for similar fixes. 

// pseudo-code
query = normalize(input)
neighbors = vectorStore.search(query, k=8)
context = summarize(neighbors)
answer = llm.generate(prompt(context, query))
return answer, diagnostics

Evaluation

Retrieval quality drives outcome quality. Instrument recall@k and track human accept/reject to continuously tune.

For production, add safety rails (redaction, grounding checks), and set up a feedback loop to capture SME corrections.
